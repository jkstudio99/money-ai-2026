## 🛠️ PART 2: Daily Coding Project (3 ระดับ)

### 🟢 EASY (1-2 ชม.)

- **ชื่อ:** `PawScan CLI` — Pet Food Nutrition Analyzer ด้วย AI
- **Problem:** เจ้าของสัตว์เลี้ยงซื้อ pet food บน Shopee แต่ไม่รู้ว่า ingredient ดีจริงมั้ย — protein เพียงพอมั้ย, มี filler ซ่อนมั้ย, เหมาะกับสายพันธุ์/อายุมั้ย
- **Stack:** Python 3.12 + `anthropic` SDK (`claude-haiku-4-5`) + `rich` (pretty output) + `typer` (CLI args)
- **Core logic:**
  ```python
  import anthropic, typer
  from rich.console import Console
  from rich.markdown import Markdown

  app = typer.Typer()
  console = Console()

  @app.command()
  def analyze(
      ingredients: str = typer.Argument(..., help="Paste ingredient list from pet food label"),
      pet_type: str = typer.Option("dog", help="dog/cat"),
      age: str = typer.Option("adult", help="puppy/kitten/adult/senior"),
      breed_size: str = typer.Option("medium", help="small/medium/large")
  ):
      client = anthropic.Anthropic()
      msg = client.messages.create(
          model="claude-haiku-4-5-20251001",
          max_tokens=800,
          messages=[{"role": "user", "content": f"""
  วิเคราะห์ ingredient list ต่อไปนี้สำหรับ {pet_type} อายุ {age} ขนาด {breed_size}:
  ---
  {ingredients}
  ---
  Output ภาษาไทยแบบนี้:
  1. **คะแนนโภชนาการ:** X/10
  2. **จุดดี:** (bullet 3 ข้อ)
  3. **ข้อระวัง:** (filler, preservative, allergen ที่พบ)
  4. **เหมาะกับ {pet_type} ประเภทนี้มั้ย:** ใช่/ไม่/ปานกลาง + เหตุผล 1 ประโยค
  5. **แนะนำ:** ควรซื้อหรือมองหาตัวอื่น?
  """}]
      )
      console.print(Markdown(msg.content[0].text))

  if __name__ == "__main__":
      app()
  ```
- **ทำไมขายได้:** pet owner ไทยกว่า 10 ล้านคน ไม่รู้จะ evaluate pet food ยังไง; package เป็น `pip install pawscan` ฿149/ปี บน Gumroad; TikTok demo "สแกน ingredient Shopee pet food ด้วย AI" → viral ในกลุ่ม pet lover → traffic สูง

### 🟡 MEDIUM (1-3 วัน)

- **ชื่อ:** `LINE CRM Bot` — Auto-Extract Customer Data จาก LINE Chat → Supabase Dashboard
- **Architecture sketch:**
  ```
  LINE OA Webhook → Receive message
        ↓
  Claude API (claude-haiku-4-5) → Extract:
    - ชื่อลูกค้า (ถ้ามี)
    - intent (สอบถาม/สั่งซื้อ/complain/follow-up)
    - product ที่สนใจ
    - urgency (ด่วน/ปกติ/รอได้)
        ↓
  Supabase → upsert customer record + tag intent
        ↓
  Next.js Dashboard → แสดง CRM: customer list + intent summary + ยังไม่ตอบ
        ↓
  Alert: ถ้า intent = "สั่งซื้อ" หรือ urgency = "ด่วน" → Line Notify เตือนเจ้าของร้าน
  ```
- **Key components:**
  - `line-webhook.ts` — Vercel Edge Function รับ LINE webhook → parse message
  - `intent-extractor.ts` — Claude Haiku: extract structured JSON `{intent, product, urgency, customer_name}`
  - `crm-dashboard.tsx` — Next.js table: customer + intent badge + last message timestamp + reply status
  - `alert-engine.ts` — ถ้า intent = "purchase" → Line Notify webhook ทันที
- **ทักษะที่จะได้:** LINE Messaging API webhook, Claude structured output (JSON mode), Supabase Realtime, Next.js App Router, Vercel Edge Functions, real-time notification pipeline
